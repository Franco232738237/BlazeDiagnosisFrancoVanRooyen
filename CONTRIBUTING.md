# Contributing Guide for Interns

## Before You Start

- Read the task description carefully.
- Ask questions before changing code if the task is unclear.
- Create a new branch for your work.
- Keep your changes focused on the assigned task.

## Branch Naming

Use clear branch names:

```text
student/<your-name>/<short-task-name>
training/<your-name>/<topic>
fix/<short-bug-name>
docs/<short-doc-name>
```

Examples:

```text
student/thabo/customer-list-page
training/anele/github-workflow
docs/job-status-workflow
```

## Commit Messages

Use short, clear commit messages.

Good examples:

```text
Add customer list page starter
Document job status workflow
Fix quote approval validation message
Add security checklist notes
```

Avoid:

```text
changes
final
stuff
update
```

## Pull Requests

Every meaningful change should go through a pull request.

Your pull request must include:

- What changed
- Why it changed
- How you tested it
- Any questions for the reviewer

Do not merge your own pull request unless a mentor tells you to.

## What Not To Commit

Do not commit:

- `.env` files
- passwords
- API keys
- database credentials
- access tokens
- node_modules
- build outputs
- personal files
- large unrelated files

## When You Are Stuck

Send an update using this format:

```text
I am working on:
I expected:
What happened:
What I tried:
What I need help with:
```
