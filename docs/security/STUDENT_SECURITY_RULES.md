# Student Security Rules

## Scope

Cyber security students may only review systems, code, and local environments that they have been explicitly given permission to review.

## Not Allowed

Students must not:

- Test production systems unless explicitly authorised.
- Test third-party systems.
- Attempt password guessing.
- Attempt social engineering.
- Access data outside their assigned scope.
- Share vulnerabilities publicly.
- Commit secrets or credentials.

## Allowed Student Security Work

Students may:

- Review code for access-control risks.
- Review API contracts.
- Review `.gitignore` and environment templates.
- Review customer data exposure risks.
- Prepare security checklists.
- Review local demo environments.
- Document findings professionally.

## Reporting Format

Use this structure:

```markdown
# Security Finding

## Area Reviewed

## Risk Level
Low / Medium / High

## Finding

## Why It Matters

## Recommended Fix

## Evidence

## Notes
```
