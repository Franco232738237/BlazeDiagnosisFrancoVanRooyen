# Remote Internship Plan

## Format

The internship is remote. Because students have different schedules, including part-time and full-time commitments, the programme will not rely on one fixed meeting that everyone must attend.

Instead, the internship will use:

- Informal virtual discussions when available
- Written updates
- GitHub issues and pull requests
- Shared documentation
- Short mentor feedback cycles

## Expectations

Students are expected to:

- Communicate availability clearly
- Communicate exam or study commitments early
- Read assigned training material
- Ask for help before falling behind
- Submit progress updates
- Keep work organised in GitHub
- Respect project confidentiality

## Availability

Students should provide their general weekly availability and any exam dates. This allows tasks to be assigned fairly and prevents the internship from creating an unhealthy work and study imbalance.

## Communication Rhythm

Because meetings are flexible, written communication becomes important.

Each student should provide a short update at least once per week, or more often when actively working on a task.

Recommended update format:

```text
What I worked on:
What I completed:
What I learned:
What I am stuck on:
What I need help with:
My next availability:
```
