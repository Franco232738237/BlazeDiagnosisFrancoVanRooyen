# First Week Blaze Diagnostics Tech Stack Training

## Purpose

The first week is mainly for training, setup, and understanding the existing Blaze Diagnostics application.

## Stack

The current repository uses:

- TypeScript
- Next.js
- Tailwind CSS
- PostgreSQL
- Prisma currently in the backend
- GitHub for version control

Earlier planning mentioned Drizzle ORM. Mentors must confirm whether the app will continue with Prisma or later migrate to Drizzle.

## Day 1: GitHub and Project Orientation

- Read `STUDENT_START_HERE.md`.
- Read `README.md`.
- Confirm your GitHub username.
- Read the project purpose.
- Review the folder structure.
- Do not start implementation work yet.

## Day 2: Local Setup and Environment

- Install Git.
- Install Node.js and npm.
- Install Visual Studio Code.
- Review `.env.example` files.
- Attempt local setup.
- Document any errors.

## Day 3: Frontend Stack

- Study TypeScript basics.
- Study React components.
- Study Next.js routing and pages.
- Study Tailwind CSS utilities.
- Find existing pages/components in the app.

## Day 4: Backend, Database, and Workflow

- Review backend structure.
- Review Prisma usage.
- Study PostgreSQL basics.
- Study entities: users, workshops, customers, vehicles, jobs, quotes, parts, invoices.
- Review MVP user stories.

## Day 5: Starter Task Preparation

- Choose or receive a first starter issue.
- Create a branch.
- Make a small change or documentation update.
- Commit clearly.
- Push branch.
- Open a pull request if instructed.

## Required Weekly Output

Each student must submit:

- What they set up
- What they learned
- What failed or blocked them
- Questions for the mentor
- Availability for the next week
