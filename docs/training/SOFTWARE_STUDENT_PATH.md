# Software Developer / Software Engineer Learning Path

## Week 1 Focus

- GitHub workflow
- Repository structure
- TypeScript fundamentals
- React and Next.js basics
- Tailwind CSS basics
- PostgreSQL and ORM basics
- Understanding existing modules

## Priority Files to Review

- `README.md`
- `STUDENT_START_HERE.md`
- `frontend/src/app/page.tsx`
- `frontend/src/components/`
- `frontend/src/features/`
- `backend/src/modules/customers/`
- `backend/src/modules/vehicles/`
- `backend/src/modules/jobs/`
- `backend/src/modules/quotes/`
- `backend/prisma/schema.prisma`
- `docs/api-contracts.md`
- `docs/database-schema.md`

## Beginner-Friendly Starter Tasks

- Document a user flow.
- Improve README setup instructions.
- Create a simple frontend status card.
- Add a placeholder page for a feature.
- Add validation messages to a form.
- Review API contracts and identify missing fields.
- Create a small mock component using Tailwind.

## Intermediate Starter Tasks

- Build customer list UI.
- Build job card detail UI.
- Add quote status display.
- Add customer-visible job timeline UI.
- Add simple backend route placeholder.
- Add validator for a DTO.
- Write manual test notes for a workflow.

## What Good Work Looks Like

- Small, focused pull requests
- Clear naming
- No secrets committed
- Acceptance criteria followed
- Questions documented
- Work tested where possible
