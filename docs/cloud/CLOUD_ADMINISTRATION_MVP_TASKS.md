# Cloud Administration MVP Tasks

## Purpose

Cloud Administration students support the project by helping plan access, environments, deployment, configuration, and operational readiness.

## Task Area 1: GitHub Organisation and Access

- List required repositories.
- Recommend access levels by group.
- Draft branch protection recommendations.
- Recommend review rules for pull requests.
- Document how students should request access.

## Task Area 2: Environment Variables and Secrets

- Review `.env.example` files.
- List required variables.
- Identify which values must never be committed.
- Draft safe secrets-handling notes.
- Confirm `.env` is ignored.

## Task Area 3: Deployment Environment Planning

Prepare notes for:

- Development environment
- Staging environment
- Production environment

For each environment, identify:

- Purpose
- Who should have access
- Database requirements
- Environment variables
- Deployment trigger
- Backup requirements

## Task Area 4: Database Hosting and Backup

- Research PostgreSQL hosting options.
- Draft backup expectations.
- Draft restore testing expectations.
- Document data retention questions.

## Task Area 5: Monitoring and Logging

- Identify what should be monitored.
- Identify what should be logged.
- Recommend basic uptime checks.
- Recommend basic error monitoring.

## First Deliverables

- GitHub access policy draft
- Environment variable checklist
- Deployment-readiness checklist
- Backup and restore planning notes
- Monitoring checklist
